# 儿童视力管理后台 — 宝塔面板部署步骤

本文档说明如何将本项目构建后的静态资源部署到服务器，并通过 **宝塔面板** 配置 Nginx 站点。部署完成后，通过域名即可访问后台，并**联接到微信云开发（CloudBase）后端**。

> **与云开发对接**：后台要能联机接到微信云开发，需先按 **[《外部后台对接 CloudBase 云函数（操作指南）》](./docs/外部后台对接CloudBase云函数_操作指南.md)** 在 Vue 项目中完成：CloudBase SDK 匿名登录、`getCloudbaseAccessToken`、`callAdminManager` 封装及登录接口改为走 `admin_manager`。部署到宝塔后，同一套逻辑会在生产环境生效，只需确保下面「配置生产环境变量」和「安全来源/安全域名」正确即可。

---

## 一、部署前准备

### 1.1 服务器与宝塔
- 已有一台 **Linux 服务器**（CentOS / Ubuntu / Debian 等）。
- 已安装 **宝塔面板**（如未安装，可到 https://www.bt.cn/new/download.html 按系统选择安装命令）。
- 能通过浏览器访问宝塔面板（如 `http://你的服务器IP:8888`）。

### 1.2 域名（可选但推荐）
- 已有一个域名（例如 `admin.yourdomain.com`）并解析到该服务器 IP。
- 若暂时没有域名，也可用 **http://服务器IP** 访问（需在宝塔中放行 80 端口）。

### 1.3 联接到微信云开发（必读）

部署到宝塔后，后台仍通过 **CloudBase HTTP 网关** 调用云函数 `admin_manager`，与本地开发是同一套逻辑。请确认：

- 已在 Vue 项目中按 **[《外部后台对接 CloudBase 云函数（操作指南）》](./docs/外部后台对接CloudBase云函数_操作指南.md)** 完成：
  - 安装 `@cloudbase/js-sdk`，匿名登录获取 `access_token`
  - 封装 `getCloudbaseAccessToken()` 与 `callAdminManager()`
  - 登录接口改为调用 `apiAdminLogin`（即走 `admin_manager`）
- CloudBase 控制台已开启 **匿名登录**（若采用对接指南中的匿名登录方式）。
- 下面 1.4 中的 **生产环境变量** 和 **安全来源/安全域名** 配置正确，否则部署后请求会 401 或跨域。

### 1.4 配置生产环境变量
在**你本机的项目目录**中编辑 **`.env.production`**，填写生产环境的 **CloudBase 云函数网关地址**（与对接指南中「二、拿到 CloudBase HTTP API 地址」一致，不要填成别的 HTTP 触发 URL）：

```env
# 应用部署基础路径（部署在根目录保持 /）
VITE_BASE_URL = /

# CloudBase 云函数网关地址（必填）- 形如 https://cloud1-xxx.api.tcloudbasegateway.com/v1/functions/admin_manager
VITE_CLOUDBASE_HTTP_URL = https://cloud1-xxx.api.tcloudbasegateway.com/v1/functions/admin_manager

# 生产去掉 console
VITE_DROP_CONSOLE = true
```

保存后，在本地执行一次构建（见下文「二、本地构建」）。  
**注意**：云开发控制台里，需在「**安全**」或「**环境配置 → 安全配置**」的 **安全来源/安全域名** 中添加你的**后台站点域名**（如 `https://admin.yourdomain.com`），否则浏览器会报跨域错误。

---

## 二、本地构建

在你本机（Windows）项目目录下执行：

```bash
cd c:\Users\Administrator\Desktop\art-lnb-master
pnpm build
```

构建成功后，会在项目根目录生成 **`dist`** 文件夹，里面包含：
- `index.html`
- `assets/` 目录（JS、CSS、图片等）

后续步骤都是把 **`dist` 里的全部内容** 放到服务器网站根目录。

---

## 三、宝塔面板 — 添加网站

### 3.1 登录宝塔
- 浏览器打开：`http://你的服务器IP:8888`（或你改过的端口）。
- 输入宝塔账号密码登录。

### 3.2 添加站点
1. 左侧菜单点击 **「网站」**。
2. 点击 **「添加站点」**。
3. 填写：
   - **域名**：例如 `admin.yourdomain.com`（用 IP 访问则填 `服务器IP`）。
   - **根目录**：选一个目录作为网站根目录，例如：
     ```text
     /www/wwwroot/admin.yourdomain.com
     ```
     宝塔会自动创建该目录。
   - **FTP**、**数据库**：本项目是纯前端静态，可不创建，直接留空或取消勾选。
   - **PHP 版本**：选「纯静态」或任意一个均可（Nginx 只负责把请求交给静态文件）。
4. 点击 **「提交」**，站点创建完成。

### 3.3 确认网站根目录路径
记下你设置的**网站根目录**（例如 `/www/wwwroot/admin.yourdomain.com`），后面要把 `dist` 里的内容上传到这个目录下，使 `index.html` 在该目录下可直接被访问。

---

## 四、上传 dist 到服务器

任选一种方式，把**本机 `dist` 文件夹内的所有内容**（不是 dist 文件夹本身）上传到上面说的**网站根目录**。

### 方式 A：宝塔文件管理器上传（适合小项目）
1. 宝塔左侧 **「文件」**。
2. 进入你站点的根目录（如 `/www/wwwroot/admin.yourdomain.com`）。
3. 删除该目录下宝塔自动生成的默认文件（如 `index.html`、`404.html` 等，若有）。
4. 点击 **「上传」**：
   - 可先在本机把 `dist` 打成 zip（只打包 dist 里的内容：选中 dist 内所有文件 → 右键压缩为 zip）。
   - 上传该 zip，上传完成后在宝塔里 **「解压」** 到当前目录。
5. 解压后确保根目录下有 **`index.html`** 和 **`assets`** 文件夹。

### 方式 B：本机用 SFTP 工具上传（如 WinSCP、FileZilla）
1. 在宝塔 **「安全」** 或 **「SSH」** 中确认 22 端口已放行；在 **「面板设置」** 或服务器里查看 SSH 账号密码（或密钥）。
2. 用 WinSCP / FileZilla 连接服务器：
   - 协议：SFTP
   - 主机：服务器 IP
   - 端口：22
   - 用户名/密码：服务器 SSH 登录账号
3. 进入站点根目录（如 `/www/wwwroot/admin.yourdomain.com`）。
4. 把本机 **`dist` 目录下的全部内容**（`index.html`、`assets` 等）上传到该目录，不要多一层 `dist` 目录。

### 方式 C：服务器上拉代码再构建（若你代码在 Git）
1. 宝塔 **「终端」** 或 SSH 登录服务器。
2. 安装 Node.js（宝塔「软件商店」搜 Node 版本管理器安装，或自行安装）。
3. 进入一个目录，克隆仓库后安装依赖并构建：
   ```bash
   git clone 你的仓库地址
   cd art-lnb-master
   npm install -g pnpm
   pnpm install
   # 在服务器上编辑 .env.production 填写 VITE_CLOUDBASE_HTTP_URL
   pnpm build
   ```
4. 把 `dist` 里的内容复制到网站根目录，例如：
   ```bash
   cp -r dist/* /www/wwwroot/admin.yourdomain.com/
   ```

## 五、配置 Nginx（支持 Vue 路由 history 模式）

前端是单页应用（SPA），刷新或直接访问子路径（如 `/vision-admin/users`）时，需要 Nginx 把请求交给 `index.html`，否则会 404。

### 5.1 在宝塔里改 Nginx 配置
1. 宝塔左侧 **「网站」** → 找到你刚添加的站点 → 点击 **「设置」**。
2. 左侧选 **「配置文件」**（或「Nginx 配置」）。
3. 在 `server { ... }` 里找到 `location / { ... }` 块，改成下面这样（保留你已有的 `root`、`server_name` 等）：

```nginx
server {
    listen 80;
    server_name admin.yourdomain.com;   # 改成你的域名或 IP
    root /www/wwwroot/admin.yourdomain.com;  # 改成你的网站根目录
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 可选：禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

说明：
- **try_files $uri $uri/ /index.html;**  
  先找真实文件，找不到再找目录，再找不到就返回 `index.html`，这样前端路由才能接管。
- **静态资源缓存**  
  `js|css|...` 那段可减少重复请求，可选。

4. 点击 **「保存」**，再在 Nginx 设置里点 **「重载配置」**（或「服务」→「重载」）。

---

## 六、HTTPS（推荐）

若使用域名，建议开启 SSL，避免浏览器提示不安全，也避免部分接口要求 HTTPS。

### 6.1 宝塔申请免费证书
1. **网站** → 该站点 **「设置」** → **「SSL」**。
2. 选 **「Let's Encrypt」**，勾选你的域名，点击 **「申请」**。
3. 申请成功后，可开启 **「强制 HTTPS」**，这样访问 http 会自动跳转到 https。

### 6.2 安全来源 / 安全域名
在 CloudBase 控制台「**安全**」或「**环境配置 → 安全配置**」里，把后台访问域名加入 **安全来源/安全域名**，例如：
- `https://admin.yourdomain.com`

---

## 七、放行端口与防火墙

- **80**：HTTP（若用 HTTPS 可只对外开 80/443，由 Nginx 处理）。
- **443**：HTTPS（若已开 SSL）。
- 宝塔 **「安全」** 里放行上述端口；若用云服务器，在云厂商控制台 **安全组** 里也放行 80、443。

---

## 八、验证部署

1. 浏览器访问：`https://admin.yourdomain.com`（或你用的域名/IP）。
2. 应出现**登录页**；用 `users` 里 `is_admin=true` 的账号（手机号+密码）登录。
3. 登录后应看到 **「儿童视力管理」** 菜单及子页面。
4. 若接口报错或跨域错误：检查 `.env.production` 里 `VITE_CLOUDBASE_HTTP_URL` 是否正确，以及控制台「安全来源/安全域名」是否包含该域名。

---

## 九、后续更新部署

以后只改了前端代码时：
1. 本机修改代码，确认 `.env.production` 无误。
2. 本机执行 `pnpm build`。
3. 按「四、上传 dist」的方式，用**新生成的 dist 内容覆盖**服务器上网站根目录里的旧文件（或先清空根目录再上传新内容）。

---

## 十、步骤速查

| 步骤 | 操作 |
|------|------|
| 0 | 先按 [《外部后台对接 CloudBase 云函数》](./docs/外部后台对接CloudBase云函数_操作指南.md) 在 Vue 项目中完成云开发对接（SDK、access_token、admin_manager） |
| 1 | 本机 `.env.production` 配置 `VITE_CLOUDBASE_HTTP_URL`（云函数网关地址），执行 `pnpm build` |
| 2 | 宝塔「网站」→ 添加站点，设置域名和根目录 |
| 3 | 把本机 `dist` 内全部内容上传到该根目录 |
| 4 | 站点设置 → Nginx 配置 → `location /` 改为 `try_files $uri $uri/ /index.html;`，保存并重载 |
| 5 | （推荐）SSL 申请证书并开启强制 HTTPS |
| 6 | 控制台「安全来源/安全域名」添加该站点域名（如 https://admin.yourdomain.com） |
| 7 | 浏览器访问域名，用管理员账号登录验证（此时后台已联接到微信云开发） |

按以上顺序操作即可在宝塔面板上完成从构建到访问的完整部署，并实现**后台与微信云开发后端的联机**。
