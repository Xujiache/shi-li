# 本地对接云开发 — 操作步骤

在本地跑通「后台登录 + 调云函数」，按下面顺序做即可。当前后台项目（art-lnb-master）已接好 CloudBase 网关：带 **Bearer access_token**、登录走 **admin_manager**，只需云开发侧配置 + 本地环境变量 + 安装依赖。

---

## 第一步：云开发侧准备

### 1.1 记下网关地址（无需在云函数列表点「开通 HTTP 触发」）
- 在 [CloudBase 控制台](https://tcb.cloud.tencent.com) 首页/概览查看当前环境的 **环境 ID**（形如 `cloud1-xxxxxx`）。
- 网关地址固定为：  
  `https://{环境ID}.api.tcloudbasegateway.com/v1/functions/admin_manager`  
  例如环境 ID 为 `cloud1-1gwuiims1ab650b6` 时，地址即为：  
  `https://cloud1-1gwuiims1ab650b6.api.tcloudbasegateway.com/v1/functions/admin_manager`

### 1.2 安全来源 / 安全域名（允许本地访问）
- 打开 **[环境配置 → 安全来源](https://tcb.cloud.tencent.com/dev?#/env/safety-source)**。
- 在「安全域名」处点击 **【添加域名】**，添加：
  - `localhost`（一般已默认）
  - 或 `localhost:5173`（若用 Vite 默认端口且需带端口）
- 保存后约 1～2 分钟生效，否则浏览器会跨域报错。

### 1.3 匿名登录（必开）
- 控制台 **身份认证** → **登录方式** → 开启 **匿名登录**。  
- 本地请求会先用匿名登录拿到 CloudBase **access_token**，再带 Bearer 调 admin_manager。

### 1.4 确保有一个可登录的管理员
1. 云开发控制台 → **数据库** → 打开 **users** 集合。
2. 找一条要当管理员的用户（或新建），必须包含：
   - **phone**：手机号（用来登录）
   - **password**：密码（和云函数里校验方式一致）
   - **is_admin**：设为 **true**
3. 记住**手机号**和**密码**，本地登录时用。

### 1.5 确认云函数和集合存在
- 云函数 **admin_manager** 已部署。
- 数据库至少有：`users`、`admin_sessions`。

---

## 第二步：本地项目配置

### 2.1 填云函数网关地址
在项目根目录打开 **`.env.development`**，配置（不要引号、不要多余空格）：

```env
# 云函数 admin_manager 网关地址（必填）
VITE_CLOUDBASE_HTTP_URL=https://cloud1-1gwuiims1ab650b6.api.tcloudbasegateway.com/v1/functions/admin_manager
```

把上面的地址换成你自己的环境 ID 对应的地址（见 1.1），保存。

### 2.2 安装依赖
在项目目录下执行（首次或新增依赖后）：

```bash
cd c:\Users\Administrator\Desktop\art-lnb-master
pnpm install
```

（项目已包含 `@cloudbase/js-sdk`，用于匿名登录获取 access_token。）

---

## 第三步：本地运行并登录

### 3.1 启动项目
```bash
pnpm dev
```

浏览器一般会打开 `http://localhost:5173`。

### 3.2 登录
- **手机号**：填 1.4 里管理员的 **phone**。
- **密码**：填该用户的 **password**。
- 点 **登录**。

### 3.3 看是否对接成功
- 能进后台、左侧出现 **「儿童视力管理」** 菜单及子菜单，说明**本地对接已成功**。
- 再点进「用户管理」等页面，能正常拉列表、不报错，即接口正常。

---

## 第四步：常见问题排查

| 现象 | 处理 |
|------|------|
| 提示「未配置 VITE_CLOUDBASE_HTTP_URL」 | 检查 `.env.development` 是否写了 `VITE_CLOUDBASE_HTTP_URL`，改完后**关掉 dev 再重新执行 `pnpm dev`**。 |
| 401 / MISSING_CREDENTIALS | 说明 Bearer 未带上：确认已装 `@cloudbase/js-sdk`、控制台已开**匿名登录**，且 `src/utils/cloudbase-auth.ts` 存在。 |
| 登录一直转圈或报错 | 看浏览器 **F12 → Network** 里失败请求的状态码和返回；**Console** 有无跨域或 401。 |
| 跨域 / CORS 报错 | 到控制台 **安全来源/安全域名** 添加 `localhost` 或 `localhost:5173`（见 1.2）。 |
| 提示非管理员 / 登录失败 | 检查 users 里该条 **is_admin** 是否为 **true**，**phone**、**password** 是否与云函数校验一致。 |
| 404 / 接口地址错 | 确认 `.env.development` 里的 URL 为网关地址（`...tcloudbasegateway.com/v1/functions/admin_manager`），且环境 ID 正确。 |

---

## 小结：本地对接 3 件事

1. **云开发**：记下网关地址、**安全来源/安全域名** 加 `localhost`（或 `localhost:5173`）、开 **匿名登录**，并有一个 `is_admin: true` 的用户。
2. **本地**：在 `.env.development` 里写 `VITE_CLOUDBASE_HTTP_URL=你的网关地址`，执行 `pnpm install` 后 `pnpm dev`。
3. **验证**：用管理员手机号+密码登录，能进「儿童视力管理」即对接成功。

完成以上步骤即可在本地完成对接；部署到服务器时把生产网关地址配到 `.env.production` 并构建部署即可。
