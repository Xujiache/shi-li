# CONSENSUS_questionnaire_index_refactor

## 1. 明确需求描述
- 重构 `miniprogram/pages/questionnaire/index/index` 页面，使其视觉表现与首页、数据看板页保持同一套设计语言。
- 保留原有主题色与业务交互，不改变页面功能范围。
- 页面中不得使用 emoji 表情作为图标或装饰元素。

## 2. 验收标准

### 2.1 视觉与交互
- 页面顶部使用与首页/看板页一致的蓝色渐变英雄区思路。
- 当前孩子信息卡的层级清晰，支持已绑定孩子与未绑定孩子两种状态。
- 问卷列表卡片在标题、规则、记录统计、操作按钮上有明确层次。
- 空状态、加载态与按钮风格与项目现有页面一致。

### 2.2 技术与代码
- `index.wxml` 中不再保留冗长复杂的规则三元表达式。
- `index.js` 中对问卷数据做统一格式化处理。
- 改动文件通过基本静态检查，不引入明显语法或样式问题。
- 改动后仍保持以下页面行为可用：
  - 拉取当前孩子信息
  - 拉取可填写问卷列表
  - 跳转切换孩子页
  - 跳转档案完善页
  - 跳转问卷详情页
  - 跳转问卷历史页

## 3. 技术实现共识
- 保留当前接口 `getChildren` 与 `getQuestionnaires` 调用方式。
- 在页面层新增轻量格式化方法，例如：
  - 孩子摘要文案
  - 问卷规则文案
  - 问卷统计数据
  - 列表项展示字段
- 使用现有 SVG 资源与 CSS 几何图形实现图标和装饰。

## 4. 边界限制
- 不改接口入参出参。
- 不修改问卷详情、问卷填写、填写历史页面。
- 不新增跨页面组件抽象，本次仅做当前页面内重构。

## 5. 集成方案
- 页面展示层基于现有数据源直接升级。
- 派生字段在 `questionnaire/index/index.js` 内部构建后回填到 `data.questionnaires`。
- 保持现有 tabBar、高级功能和跳转路由不变。
