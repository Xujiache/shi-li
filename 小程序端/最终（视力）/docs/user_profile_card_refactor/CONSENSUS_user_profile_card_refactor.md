# CONSENSUS_user_profile_card_refactor: 个人中心头像信息卡大厂风重构

## 1. 明确需求与验收标准

### 1.1 需求描述
- 对个人中心顶部身份信息卡进行“大厂风”视觉重构，仅限圈选区域。
- 不改变现有功能与交互逻辑。

### 1.2 验收标准（可测试）
- 仅修改 `profile card` 相关样式/结构，其他模块视觉保持不变。
- 信息层级清晰：姓名主标题、用户编号次级标签、按钮明确可点击。
- 留白与对齐统一：头像、文字与按钮纵向对齐，整体边距一致。
- 视觉克制：轻量描边与阴影，不出现强烈高亮或夸张装饰。
- 在常见屏幕宽度下不卡顿、不溢出、不截断异常。

## 2. 技术实现方案与约束

### 2.1 技术方案
- 以 WXSS 为主进行重构，必要时做最小 WXML 结构调整。
- 保持现有数据字段与事件绑定不变（如 `onEditName`, `goSelectChild`）。

### 2.2 技术约束
- 只能改动圈选区域相关代码（`profile-card` 及子元素）。
- 不新增外部依赖与资源文件。

## 3. 任务边界限制
- 不涉及页面路由、云函数、数据库或其他页面的改动。
- 不调整底部导航、快捷入口、更多菜单等区域。
