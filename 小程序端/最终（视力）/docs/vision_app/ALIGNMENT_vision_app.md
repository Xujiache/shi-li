# ALIGNMENT_vision_app: 儿童视力与体质管理小程序

## 1. 项目上下文分析

### 1.1 现有项目结构
- **类型**: 微信小程序 (云开发模式)
- **核心目录**:
  - `miniprogram/`: 小程序前端代码
  - `cloudfunctions/`: 云函数代码
- **技术栈**: WXML, WXSS, JS/TS, 微信云开发 (Cloud Functions, Cloud Database, Cloud Storage)

### 1.2 业务域与数据模型
- **核心业务**: 儿童视力与体质数据管理、预约报名。
- **用户角色**: 家长 (C端用户), 管理员 (未来Vue后台用户).
- **核心实体**:
  - 用户 (User/Parent)
  - 孩子档案 (ChildProfile)
  - 预约项目 (AppointmentItem)
  - 预约记录 (AppointmentRecord)
  - 检测记录 (CheckupRecord)
  - 轮播图 (Banner)

## 2. 需求理解与特性规范

### 2.1 核心功能模块
1.  **注册/登录**:
    - 支持手机号+密码、手机号+验证码、微信快捷登录。
    - 强制协议勾选。
    - 初始入口为登录页。
2.  **个人档案 (核心前置)**:
    - 登录后强制校验档案完善状态。
    - 包含：姓名、性别、生日、学校、班级 (必填); 身高、体重、症状、视力数据等 (选填)。
    - 支持数据自动加载与更新。
3.  **首页**:
    - 轮播图展示。
    - 预约报名入口 (下拉选择项目 -> 选择时间 -> 提交).
    - 数据看板入口 (关联孩子信息).
4.  **数据/历史记录页**:
    - 展示详细检测数据 (视力、屈光度、体质等).
    - 数据对比 (趋势箭头).
    - 结论与建议.

### 2.2 UI/UX 规范
- **配色系统**:
  - Primary: #0077C2 (100), #59a5f5 (200), #c8ffff (300)
  - Accent: #00BFFF (100), #00619a (200)
  - Text: #333333 (100), #5c5c5c (200)
  - Bg: #FFFFFF (100), #f5f5f5 (200), #cccccc (300)
- **交互流程**:
  - 登录 -> 档案校验 -> (未完善 -> 档案页) / (已完善 -> 首页).
  - 档案页提交 -> 首页.

## 3. 智能决策与假设

### 3.1 数据库设计假设
为了适配未来的Vue后台，我们将采用标准的集合结构：
- `users`: 存储登录凭证、手机号、密码(加密)。
- `children`: 存储孩子档案，关联 `_openid` 或 `user_id`。
- `banners`: 轮播图配置。
- `appointment_services`: 预约项目配置。
- `appointment_schedules`: 排班/时间段配置。
- `appointment_records`: 用户预约记录。
- `checkup_records`: 视力与体质检测记录。

### 3.2 登录逻辑决策
- 优先使用微信云开发的 `login` 云函数获取 OpenID。
- 对于“手机号+密码”注册，将在 `users` 集合存储哈希后的密码。
- 短信验证码功能需要开通云开发短信服务（假设目前开发环境模拟或使用模拟验证码，或者用户需自行配置）。为简化开发，初期可模拟验证码逻辑或侧重微信一键登录。

### 3.3 疑问与澄清
- **Q1**: “手机号+验证码”登录是否需要对接真实短信API？
  - **决策**: 开发阶段使用模拟验证码或控制台日志输出验证码，生产环境需配置腾讯云短信服务。
- **Q2**: “系统自动识别年龄”？
  - **决策**: 前端根据出生日期实时计算。

## 4. 任务边界
- 本次任务仅包含 **小程序端** 的开发。
- **后台管理** 功能仅在数据库层面预留支持，不开发Vue后台页面。
- 数据录入（如检测数据、轮播图配置）初期可通过云开发控制台直接操作数据库进行模拟，或提供简易的临时管理页（如果时间允许，否则以手动录入DB为主）。
