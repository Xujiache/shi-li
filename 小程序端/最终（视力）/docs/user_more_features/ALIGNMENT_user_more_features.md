# ALIGNMENT_user_more_features: 个人中心“更多”功能扩展

## 1. 项目上下文分析

### 1.1 目标页面
- 页面路径：`miniprogram/pages/user/index`
- 页面职责：
  - 展示当前账号与孩子档案摘要
  - 提供常用功能快捷入口
  - 提供“更多”区块承接次级功能入口

### 1.2 当前页面现状
- `个人中心`顶部资料卡、快捷入口区已经存在。
- 当前“更多”区块仅包含 2 个条目：
  - `关于与帮助`
  - `退出登录`
- 现有“关于与帮助”仅为弹窗提示，不是独立功能页。

### 1.3 已有可复用能力
- 已有页面：
  - `pages/user/appointments/index`：我的预约
  - `pages/history/list/index`：检测记录
  - `pages/dashboard/index/index`：健康看板
  - `pages/questionnaire/index/index`：问卷中心
- 已有接口：
  - `getTerms()`：可获取协议配置内容

## 2. 原始需求
- 给个人中心里面的“更多”下面增加更多的功能。

## 3. 需求理解（当前阶段）
- **优先方向**：
  - 优先接入项目中已经存在的能力，减少新增页面和后端改造风险。
  - 保持个人中心当前视觉风格与交互样式一致。
- **可扩展候选项**：
  - 问卷中心入口
  - 检测记录入口
  - 健康看板入口
  - 用户协议/隐私政策入口
  - 关于与帮助优化为更清晰的信息页或协议入口组

## 4. 决策与约束
- 保持当前小程序原生技术栈，不切换框架。
- 优先复用现有页面和现有接口，不做无规划扩展。
- 如果新增菜单项需要对应新页面，必须先确认范围再实施。
- “更多”区块新增内容需与当前信息架构一致，避免与顶部快捷入口重复过多。

## 5. 任务边界
- **包含**：
  - 个人中心“更多”区块功能扩展
  - 菜单结构和跳转逻辑调整
  - 必要的协议展示或帮助展示优化
- **暂不包含**：
  - 无现成承接能力的全新业务模块
  - 后端新接口的大范围扩展
  - 与个人中心无关的页面改造
