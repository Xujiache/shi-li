# FINAL_banner_dashboard_fix

## 1. 完成情况总结

本次交付解决了两类问题：

1) **看板页内容宽度不一致**：修复为与首页同宽（避免 flex 容器默认居中导致内容变窄）。
2) **轮播图后台可更换 + 主/副标题**：后台可编辑保存 `title/sub_title`，小程序首页轮播图可展示。

## 2. 关键实现点

- 看板页通过局部覆写 `.container { align-items: stretch; }`，覆盖全局默认样式。
- `banners` 数据模型新增 `title/sub_title`：
  - 云函数 `admin_manager` 在 create/update 中做字段校验与持久化。
  - 后台管理页增加表单项与列表展示，提交时透传字段。
  - 小程序首页轮播 WXML/WXSS 增加遮罩与文字层，有值才渲染。
- 首页 `onShow` 主动刷新轮播数据，减少“后台改了但前端看不到”的感知问题。

## 3. 质量评估

- **兼容性**：历史 banner 无标题字段不影响展示；标题为空不渲染遮罩层。
- **可维护性**：字段校验前后端一致（长度限制明确）；改动点集中、可快速回溯。
- **风险**：需确保云函数已重新部署；否则后台/小程序端可能出现字段不一致（旧云函数不接收新字段）。
