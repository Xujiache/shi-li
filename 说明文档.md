# 说明文档

## 一、项目规划

### 1.1 项目名称

儿童视力管理项目云开发迁移至 Node 后端与 MySQL

### 1.2 项目目标

1. 将 `小程序端/最终（视力）/cloudfunctions` 中的业务后端能力迁移到 `node后端（新增）`。
2. 将 `后台/art-lnb-master` 与 `小程序端/最终（视力）/miniprogram` 的接口依赖从微信云开发切换为标准 HTTP API。
3. 使用本地 MySQL 作为唯一主数据库，不迁移微信云开发旧数据，按新库新表重建业务数据。
4. 保留当前项目的核心业务能力，包括手机号登录、微信快捷登录、孩子档案、预约、检测记录、后台管理、协议配置与统计面板。
5. 新增完整问卷体系，支持高级题型、年级派发、草稿保存、后台查看与导出。

### 1.3 项目范围

纳入范围：

- `小程序端/最终（视力）/miniprogram`
- `后台/art-lnb-master`
- `node后端（新增）`
- 新建根目录 `docs/云开发迁移到Node后端/`
- 本地 MySQL 数据库初始化、表结构、种子数据、文件上传与静态资源服务

不纳入范围：

- 微信云开发旧数据迁移
- 微信云存储历史文件迁移
- 新增与现有业务无关的新模块
- 复杂 RBAC 权限系统扩展
- 多机部署、高可用、对象存储公网化等二期能力

### 1.4 端划分

| 端 | 目标用户 | 核心目标 | 当前状态 |
| --- | --- | --- | --- |
| 小程序端 | 家长/用户 | 登录、档案维护、预约、查看历史检测数据 | 已完成迁移 |
| 管理后台 | 管理员 | 用户、档案、预约、检测记录、轮播图、协议、统计管理 | 已完成迁移 |
| Node 后端 | 系统服务 | 统一承接认证、业务、文件上传、MySQL 存储 | 已完成收敛 |

### 2.3 技术路线概览

1. 小程序端通过 HTTP API 访问 `node后端（新增）`，不再调用 `wx.cloud.callFunction`。
2. 管理后台通过 REST API 访问 `node后端（新增）`，不再通过 CloudBase HTTP 网关调用 `admin_manager`。
3. Node 后端使用 Express 承接移动端与后台端两套路由，使用 MySQL 持久化业务数据。
4. 微信快捷登录改为 `wx.login -> code -> Node -> 微信官方 jscode2session`。
5. 文件存储改为 Node 本地上传目录，统一通过静态资源地址访问。

### 2.4 风险与依赖

| 类别 | 风险/依赖 | 处理方式 |
| --- | --- | --- |
| 数据库 | 需使用本地 MySQL 且不能把账号密码写入源码 | 仅写入 `.env`，代码仅读取环境变量 |
| 微信登录 | 微信快捷登录不再依赖云开发，需要补齐官方登录链路 | 在 Node 后端新增微信登录服务与配置 |
| 文件存储 | 当前前后端存在 `cloud://` 文件 ID 逻辑 | 统一替换为本地上传与 HTTP 静态访问 |
| 结构收敛 | `node后端（新增）` 存在根层与 `src/` 平行结构 | 迁移时明确单一目录结构并淘汰重复骨架 |
| 前端改造 | 小程序和后台都需要替换接口封装 | 先完成后端接口契约，再逐端改造并联调 |
