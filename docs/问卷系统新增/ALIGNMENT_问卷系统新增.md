# ALIGNMENT_问卷系统新增

## 一、原始需求

1. 小程序新增独立的问卷功能，并在小程序里单独显示。
2. 后台可以单独设置问卷内容、查看填写内容。
3. 问卷体系要完整，支持高级配置。
4. 问卷可按年级、年级范围分配。
5. 班级等字段可以配置为用户手填，或自动读取小程序孩子档案信息。
6. 后台要看到全部填写数据。

## 二、需求理解

### 2.1 新增功能本质

本次不是简单新增一张“调查表”，而是新增一套完整问卷系统，覆盖：

- 问卷模板配置
- 题目/分组配置
- 按年级/班级/孩子派发
- 草稿与正式提交
- 后台查看、筛选、导出

### 2.2 多端范围

| 端 | 范围 |
| --- | --- |
| 小程序端 | 新增独立底部 Tab `问卷`，包含问卷中心、详情、填写、历史 |
| 管理后台 | 新增 `问卷配置`、`问卷填写数据` 两个菜单 |
| Node 后端 | 新增问卷配置、派发规则、提交与答卷查看接口 |
| MySQL | 新增问卷 7 张核心表，并补孩子/班级结构化年级字段 |

## 三、任务边界

### 3.1 纳入范围

- 问卷模板、题目、选项、分组、派发规则
- 提交次数规则
- 草稿保存
- 答卷明细查看与导出
- 年级范围派发

### 3.2 不纳入范围

- 图表化问卷统计分析页
- 复杂跳题可视化编排器
- 真机微信登录联调

## 四、关键共识

- 问卷体系按“高级版”设计。
- 提交规则按问卷或派发规则配置，不写死。
- 小程序问卷入口使用新增底部 Tab。
- 年级字段使用结构化 `grade_name`，不再长期依赖 `class_name` 文本解析。

## 五、现有项目对齐

- 小程序正式代码位于 `小程序端/最终（视力）/miniprogram`
- 后台正式代码位于 `后台/art-lnb-master`
- Node 后端正式代码位于 `node后端（新增）`
- 问卷能力基于现有 Node + MySQL 主线扩展，不再依赖微信云开发
