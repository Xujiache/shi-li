# 家长 ↔ 孩子绑定 与 CRM 客户化 — 操作说明

本次修复围绕用户反馈的 3 个数据可见性问题：
1. 小程序家长建的孩子档案，员工 App 搜不到
2. 小程序注册的家长在后台「客户」栏不显示、无法分配
3. 没法验证家长 ↔ 孩子是否绑定

## 数据流（修改后）

```
小程序家长操作                   后端写入
───────────────────────────────────────────────────────
注册 / 登录           ──>  users 表 ── 自动建 ──> customers 表（待分配池）
建孩子档案            ──>  children 表 (user_id, parent_phone 已绑)
                            │
                            └─ 等管理员手动「分配部门」(child_dept_assignments) 才进员工 App
```

注意：**孩子档案不会自动出现在员工 App**。员工 App 列表严格按"孩子-部门"分配过滤；
必须由管理员后台「孩子档案 → 分配部门」操作后，对应部门的员工才能在 App 搜到。
这是设计上保留的人工权限审查环节。

## 管理员操作流程

### A. 把家长分配给员工（CRM 客户化）
1. 后台 → **客户管理**
2. 搜索区选「分配 = 待分配池」 → 看到所有小程序注册自来但还没分配的家长
3. 点行尾「编辑」 → 选「归属员工」 → 保存
4. 系统会自动推送通知给该员工 App

### B. 把孩子档案分配给部门
1. 后台 → **孩子档案**
2. 找到该孩子（可按家长手机号搜） → 点「分配部门」
3. 勾选可访问该档案的部门 → 保存
4. 该部门员工的 App「孩子」tab 立即可见

### C. 验证「家长 ↔ 孩子」绑定
有 3 个入口可看：
- 后台 **用户管理** → 编辑某家长 → 弹窗底部「名下孩子档案」表 + 「客户档案」摘要
- 后台 **孩子档案** 列表的「家长编号 / 家长昵称 / 家长手机号」3 列
- 员工 App **客户详情** → 切到「孩子档案」tab（前端已实现，前提是该家长已分配给当前员工）

## 老数据 backfill

修复上线前已经注册过、但还没在 `customers` 表的家长，**只需要在小程序重新登录一次**，
后端会自动建客户档案（`syncMobileCustomer` 在 register/login/wechat-login 三处都会调）。

如果不想等家长自己登录，可手工 SQL backfill：

```sql
INSERT INTO customers (
  customer_no, user_id, display_name, phone, gender,
  source, status, level, tags, active, next_follow_up_text, remark
)
SELECT
  CONCAT('C', DATE_FORMAT(NOW(),'%Y%m%d'), LPAD(FLOOR(RAND()*10000),4,'0')) AS customer_no,
  u.id, COALESCE(NULLIF(u.display_name,''), u.phone, '家长'), u.phone, 'unknown',
  'miniprogram', 'potential', 'C', JSON_ARRAY(), 1, '', ''
FROM users u
WHERE u.deleted = 0
  AND NOT EXISTS (SELECT 1 FROM customers c WHERE c.user_id = u.id AND c.active = 1);
```

⚠️ `customer_no` 可能 UNIQUE 冲突（生成的 4 位序号巧合相同）。冲突就重跑这条 SQL，
直到 affected_rows = 0。

## 测试要点

1. **小程序新用户注册** → 后台「客户管理 → 待分配池」立即看到该家长
2. **后台分配员工** → 员工 App 客户列表立即出现 + 收到通知
3. **小程序建孩子档案** → 后台「孩子档案」列表立即看到，但员工 App「孩子」tab 仍然没有
4. **后台给孩子分配部门** → 对应部门员工 App「孩子」tab 立即能看到
5. **员工 App 打开该客户详情 → 孩子档案 tab** → 显示该家长名下所有孩子
6. **后台编辑该家长** → 弹窗底部能看到名下孩子表 + 客户档案信息（含归属员工）
7. **老家长重登一次** → 后台「客户管理 → 待分配池」出现该家长（幂等：重登多次只建一次）
