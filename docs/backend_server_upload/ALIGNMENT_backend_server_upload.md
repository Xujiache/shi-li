# ALIGNMENT_backend_server_upload: Node 后端上传服务器

## 1. 项目上下文分析

### 1.1 目标范围
- 目标模块：`node后端（新增）`
- 目标动作：
  - 将 Node 后端代码上传到 Ubuntu 24.04.1 LTS 服务器
  - 为后续数据库初始化、环境配置和服务启动做准备

### 1.2 当前环境信息
- 服务器系统：Ubuntu 24.04.1 LTS amd64
- 服务器 IP：`154.36.160.34`
- 登录用户：`root`
- 当前暂无域名、暂无备案
- 当前暂无现成 MySQL 库

### 1.3 已确认的本地能力
- 本机存在：
  - `ssh`
  - `scp`
  - `plink`
  - `pscp`
  - `python`
- 已读取项目说明文档，确认当前项目为：
  - 小程序端
  - 管理后台
  - Node 后端

## 2. 原始需求
- 帮用户把后端上传到服务器。

## 3. 需求理解（当前阶段）
- **本次优先目标**：
  - 建立到服务器的可用 SSH 连接
  - 将 `node后端（新增）` 上传到服务器指定目录
- **后续顺延动作**：
  - 安装依赖
  - 初始化 MySQL
  - 配置 `.env`
  - 启动 Node 服务

## 4. 当前阻塞问题
- 已使用以下方式测试 SSH：
  - `plink`
  - OpenSSH `ssh -vvv`
  - `ssh-keyscan`
  - Python 原始 socket 读取 22 端口 banner
- 现象一致：
  - TCP 22 端口可连通
  - 但 SSH 握手初期即被远端关闭或未返回标准 SSH banner
- 结论：
  - 当前服务器 SSH 服务不可正常完成握手，尚无法登录，也无法上传代码

## 5. 决策与约束
- 在 SSH 未恢复前，不进行伪上传或假设成功操作。
- 遇到服务器连接阻塞，按规范中断并记录问题，等待人工修复后继续。
- 本次不扩展到域名和小程序正式上线，仅聚焦“后端上传服务器”。

## 6. 需用户处理的关键事项
- 修复服务器 SSH 服务，使其可以正常接受标准 SSH 登录。
- 或提供可用的替代连接方式：
  - 正常 SSH 端口
  - 跳板机
  - 面板文件管理/控制台
  - SFTP/FTP
