## 三、技术债

### 3.1 后端

- [ ] [scripts/data/users.js](../../node后端（新增）/scripts/data/users.js) 是历史种子脚本，引用了不存在的字段（username / role_id），与当前 schema 不匹配；建议清理或重写
- [ ] DB 变更目前用 `CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 模式，**没有正式 migration 工具**；客户数 / 业务复杂度上升后需引入 [knex](https://knexjs.org/) / [umzug](https://github.com/sequelize/umzug) 等工具
- [ ] 审计日志表 [audit_logs](../../node后端（新增）/scripts/db/core.js) 数据增长会快，建议 v1.1 做按月分表或冷数据归档

### 3.2 员工 App

- [ ] UniApp 在低端 Android 设备首次启动时间偏长（> 2s），需 v1.1 优化首屏 chunk
- [ ] 大列表（200+ 条客户）滚动需引入虚拟列表（uview-plus 不带，可考虑 z-paging）
- [ ] 本地 SQLite 在 H5 端不可用，目前 fallback 到强制在线模式；需评估 IndexedDB 实现成本
- [ ] 离线照片堆积可能撑爆沙盒；v1.1 引入分批上传 + 上传后立即删本地原图

### 3.3 PC 管理后台

- [ ] 客户全局列表当跨员工多筛选 + 大数据时表格性能压力大；考虑虚拟滚动或服务端 cursor 分页
- [ ] 公告富文本编辑器选型偏简陋（建议 wangEditor 5.x），后期升级时注意富文本字段兼容性
- [ ] [art-lnb-master](../../后台/art-lnb-master/) 的构建产物较大（dist 目录），需配合 [DESIGN_员工app](DESIGN_员工app.md) 优化（v1.1）
