## 详细用例

### NCH-001 · APK 安装与冷启动 ≤ 2s
- **步骤**：
  1. 安装签名 APK（包内含 employee H5 资源）。
  2. 完全杀进程；点击图标启动并秒表计时到首屏可交互。
  3. 重启手机后再次冷启动。
- **预期**：两次冷启动均 ≤ 2s；启动期间无白屏 > 800ms。
- **平台**：Android 5 / 8 / 10 / 13+。
- **对应**：§4.4 冷启动。

### NCH-002 · staff 首登 → 强制改密 → 工作台
- **步骤**：
  1. 用预置 staff 账号 + 初始密码登录。
  2. 应被强制跳改密页（不能跳过）。
  3. 提交新密码后跳工作台。
- **预期**：跳转链路无错；改密后下次冷启动直接到工作台；tabBar 5 项中 "工作台" 默认选中。
- **平台**：全部。
- **对应**：§4.1 首次登录强制改密。

### NCH-003 · manager 首登 → 改密 → 看到"待审批"快捷入口
- **步骤**：用 manager 账号执行 NCH-002，进入工作台后查 4 数据卡 + 公告 + "待审批" 入口。
- **预期**：manager 工作台明显比 staff 多 "待审批" 入口；进入后看到自己组的转出申请。
- **平台**：全部。
- **对应**：§4.1 角色感知 tabBar 切换正确 / 4 数据卡 + 公告。

### NCH-004 · 点客户手机号直接拨号
- **步骤**：客户详情页点手机号；微信内打开 H5 时也走查一次。
- **预期**：Android 拉起拨号面板；iOS 拉起 FaceTime / 拨号；微信内 H5 弹"是否打电话"确认框。
- **平台**：Android / iOS / 微信内核。
- **对应**：§4.1 点手机号直接拨打；注意 iOS Safari `format-detection: telephone=no` 已设，需要 `<a href="tel:">` 显式触发。

### NCH-005 · 飞行模式登录（7 天内本地缓存）
- **步骤**：
  1. 在线登录一次，确认本地 token 落库。
  2. 开飞行模式。
  3. 杀进程冷启动，再次进入登录页 → 输入相同账号密码。
- **预期**：本地缓存命中，进入工作台；状态栏 / 顶部出现 "离线模式" 提示。
- **平台**：Android / iOS。
- **对应**：§4.2 飞行登录。

### NCH-006 · 飞行模式新建客户
- **步骤**：飞行下点 "新建客户" → 填基本字段 → 保存。
- **预期**：本地立即生成 client_uuid；客户列表能立刻看到该条（带 "未同步" 角标 / pending 标识）；DB 中 `pending_op` 表 +1 条 INSERT。
- **平台**：Android / iOS。
- **对应**：§4.2 飞行新建。

### NCH-007 · 飞行模式写跟进 + 拍 3 张照
- **步骤**：飞行下进客户详情 → "写跟进" → 输入文字 → 点附件 → `uni.chooseImage` 拍 3 张。
- **预期**：3 张图本地落 sandbox；跟进列表立刻可见；pending_op 表 +1 跟进 +3 附件，state=pending。
- **平台**：Android / iOS。
- **对应**：§4.2 离线跟进 + 附件。

### NCH-008 · 飞行模式设跟进提醒
- **步骤**：写跟进时勾 "设提醒" → 选时间（5 分钟后）→ 保存 → 等到点。
- **预期**：到点本地推送响起；点通知能跳到该跟进；联网后服务端也产生站内消息。
- **平台**：Android / iOS。
- **对应**：§4.2 离线提醒；§4.1 跟进提醒。

### NCH-009 · 网络恢复 30 秒内自动 sync
- **步骤**：完成 NCH-006 ~ NCH-008 → 关闭飞行模式 → 秒表计时。
- **预期**：30 秒内出现同步 toast；pending_op 表清空；新建客户、跟进、附件全部在云端可见。
- **平台**：Android / iOS。
- **对应**：§4.2 自动同步。

### NCH-010 · 同 client_uuid 重复提交 → duplicate
- **步骤**：用 Postman / curl 把 sync 请求体（含相同 client_uuid）连续 POST 两次。
- **预期**：第一次 200 created；第二次 200 status=duplicate；服务端只生成一条；audit 日志可追溯。
- **平台**：任意（接口测试，但要在真机环境复现 client_uuid 是否生成稳定）。
- **对应**：§4.2 幂等去重。

### NCH-011 · 附件上传失败重试
- **步骤**：飞行模式下拍照 → 解除飞行模式但弱网 / 主动 kill 上传请求 → 触发 3 次重试。
- **预期**：3 次失败后该附件标红 + "手动重传" 按钮；点后能成功；UI 不阻塞其他同步。
- **平台**：Android / iOS。
- **对应**：§4.2 附件重传。

### NCH-012 · 跨设备冲突合并
- **步骤**：
  1. A 机离线改客户备注为 "A-备注"；B 机离线改同一客户为 "B-备注"。
  2. 先让 A 机联网同步，再让 B 机联网。
- **预期**：B 机弹冲突合并 UI；展示双方版本；员工选择保留 A / B / 合并文本；操作后服务端只剩一条最终值。
- **平台**：双机（Android × Android / iOS × Android 任意组合）。
- **对应**：§4.2 冲突合并。

### NCH-013 · 切账号 pendingCount > 0 弹 modal
- **步骤**：账号 X 飞行下产生 pending_op；不同步直接点 "切换账号" 登录账号 Y。
- **预期**：弹 modal 三选一：
  - "先同步"（默认；提示需联网）
  - "放弃本地变更并切换"
  - "取消"
  无论选哪个都不会丢账号 X 的数据（除非主动选放弃）。
- **平台**：Android / iOS。
- **对应**：§4.2 pending_op 强校验。

### NCH-014 · APK 一周存储 ≤ 80MB
- **步骤**：单设备连续使用 7 天（每天约 30 客户、10 跟进、5 附件）。
- **预期**：第 7 天 "设置 → 应用 → 存储" 显示 ≤ 80MB；包含 sandbox + sqlite + 缓存。
- **平台**：Android / iOS。
- **对应**：§4.4 单设备存储。

### NCH-015 · 微信内置浏览器 H5 加载
- **步骤**：微信内打开 `http://<域名>/employee/`（在线模式），走查登录 / 工作台 / 客户列表。
- **预期**：
  - 页面正常加载；
  - 底部 tabBar 不被微信底栏遮挡；
  - 点 tel: 链接弹微信确认拨号；
  - 视频 / 音频自动播放被禁，需用户点击触发；
  - JSSDK 不需调用（H5 页面无微信能力依赖）。
- **平台**：微信 Android（X5）+ 微信 iOS（WKWebView）。
- **对应**：§4.3 H5-微信内置；详见 `safari-wechat-notes.md`。

### NCH-016 · iOS Safari + Chrome 在线模式 H5 走查
- **步骤**：在 iOS 14 / 16 / 17 真机分别用 Safari 和 Chrome iOS 打开 H5。
- **预期**：
  - 视图无 100vh 抖动（Safari 顶部地址栏隐藏 / 显示）；
  - 输入框聚焦时不缩放；
  - 安全区适配（刘海屏）：tabBar 距底部留出 home indicator 空间；
  - hash 路由刷新不报 404。
- **平台**：iOS 14 / 16 / 17。
- **对应**：§4.3 H5-Safari。

### NCH-017 · Android 5 旧 WebView 兼容
- **步骤**：Android 5（API 21）真机 + 系统 WebView 极老版本（不更新 GMS）打开 H5 / 装 APK 启动。
- **预期**：
  - APK 走 X5 内核，H5 全部能渲染；
  - 系统 WebView 模式下若白屏，记录浏览器版本与 console error，但 Phase 4 仅要求 APK 能跑。
- **平台**：Android 5。
- **对应**：§4.3 第 1 行（Android 5）。

### NCH-018 · 异地登录触发短信二次校验
- **步骤**：在登记 IP 之外（如 4G + 异地 SIM）登录已激活账号。
- **预期**：登录后弹短信验证码；填错 3 次锁定 5 分钟；填对继续。
- **平台**：真 SIM 卡 Android / iOS。
- **对应**：§4.1 异地登录。

### NCH-019 · 单设备限制
- **步骤**：A 机登录 staff；B 机用同账号登录。
- **预期**：B 机登录成功；A 机下次任意 API 调用 401，强制踢回登录页并 toast "您的账号已在其他设备登录"。
- **平台**：双机。
- **对应**：§4.1 单设备限制。

### NCH-020 · 客户列表 200 条滚动流畅
- **步骤**：登录到至少有 200 条客户的账号；快速上下滑动客户列表。
- **预期**：60 fps 体感流畅，无明显掉帧 / 白屏块；图片懒加载到位。
- **平台**：Android 8 + / iOS 14 +。
- **对应**：§4.4 滚动流畅。
