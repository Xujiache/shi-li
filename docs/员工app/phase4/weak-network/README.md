## 为什么本机不能跑

Phase 4 实验室阶段在 Linux 服务器跑，无 chromium / 无 HBuilderX / 无真机网络。"弱网"必须在以下条件下验证：

- 物理设备的真实蜂窝网络（4G / 5G / 信号 1 格）
- 不同网络间的切换（4G ↔ WiFi / 飞行模式 ↔ 数据 / 信号有→无→有）
- 系统级丢包 / 高延迟（不是 loopback `tc qdisc` 模拟，因为后端在同机，模拟无意义）

本机仅做了**性能压测正常网络下的基线**（[../perf/summary.md](../perf/summary.md)），未模拟弱网。

## 真机阶段需要执行的弱网测试用例

### 1. 信号 1 格场景

执行人：测试同事
设备：Android 真机（有蜂窝网卡）
工具：手机自带网络切换 + 信号屏蔽箱（如有）/ 进入电梯 / 地下车库

| 编号 | 用例 | 预期 |
|---|---|---|
| WN-01 | 信号 1 格下登录 | 5 秒内成功，未触发 timeout |
| WN-02 | 信号 1 格下加载客户列表 200 条 | 8 秒内显示，分批加载 |
| WN-03 | 信号 1 格下提交跟进（含 1 张照片） | 提交按钮立即 toast"已提交"，照片在后台懒上传 |
| WN-04 | 信号 1 格下 sync/batch 100 ops | 不出现 timeout（90s 限）；同步成功率 ≥ 95% |

### 2. 网络切换场景

| 编号 | 用例 | 预期 |
|---|---|---|
| WN-05 | 写跟进过程中 4G → WiFi | 不丢失草稿，sync 自动重新触发 |
| WN-06 | sync 进行中网络断开（飞行模式开） | 当前批处理标 failed，pending_op 状态保留，next_retry_at 设为退避 |
| WN-07 | 飞行模式 → 数据网络（断转通）| `uni.onNetworkStatusChange` 触发，30 秒内自动 sync.kick |
| WN-08 | 切换 WiFi（不同热点）| token 不失效，sync 继续 |

### 3. 丢包 / 高延迟

| 编号 | 用例 | 预期 |
|---|---|---|
| WN-09 | 模拟 30% 丢包（信号差）下提交 100 条 op | 重试 5 轮后整体成功率 ≥ 95% |
| WN-10 | 模拟 RTT 2s+ 下登录 + 主流程 | 每个接口 timeout 设置（http.ts 默认 15s）足够；不出现 hang |

### 4. 离线 → 在线 端到端

| 编号 | 用例 | 预期 |
|---|---|---|
| WN-11 | 飞行模式登录（用 7 天内本地缓存）| auth.bootstrap 直读本地 token，进入工作台 |
| WN-12 | 飞行模式新建客户 + 写跟进 + 拍 3 张照 | 全部本地写入 + pending_op 排队 |
| WN-13 | 关闭飞行模式后 30s 内自动 sync | sync-indicator 由红→黄→绿，pending 数清零 |
| WN-14 | 长时间断网（30 分钟）后恢复 | retry_count 不超过 5；恢复后立即 sync |
