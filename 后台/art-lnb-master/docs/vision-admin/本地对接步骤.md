# 本地对接云开发 — 操作步骤

在本地跑通「后台登录 + 调云函数」，按下面顺序做即可。

---

## 第一步：云开发侧准备

### 1.1 拿到 admin_manager 的网关地址（HTTP 触发）

环境 ID 已知为：`cloud1-1gwuiims1ab650b6` 时，可直接使用 **API 网关** 地址（无需在控制台单独“开通 HTTP 触发”）：

```text
https://cloud1-1gwuiims1ab650b6.api.tcloudbasegateway.com/v1/functions/admin_manager
```

若控制台有 **「接入指引」**：在云函数列表里点 **admin_manager** 所在行的 **「接入指引」**，查看 **HTTP API** 的调用地址，与上面格式一致即可。

### 1.2 安全来源 / 安全域名（必配，否则浏览器会跨域报错）

前端是浏览器页面，请求会发到 `*.api.tcloudbasegateway.com`，必须在云开发里配置 **安全域名**：

1. 打开 **云开发控制台**（微信开发者工具 → 云开发，或腾讯云 CloudBase 控制台）。
2. 进入 **「安全」** 或 **「环境配置 → 安全配置」**。
3. 在 **Web 安全域名** / **安全来源** 中新增：
   - 本地开发：`http://localhost:3006`（与项目 `.env` 里 `VITE_PORT=3006` 一致；若改过端口则填 `http://localhost:你的端口`）
   - 生产环境：你的后台前端域名，如 `https://admin.yourdomain.com`

保存后，浏览器才能正常请求云函数网关，否则会报跨域错误。

### 1.3 开启匿名登录（用于网关 Bearer 认证）

本后台通过 **匿名登录** 获取 CloudBase 的 `access_token`，再携带该 token 调用 HTTP 网关。请在控制台：

- 进入 **「登录授权」** 或 **「身份认证」**。
- 开启 **匿名登录**。

未开启时，前端会报「获取 CloudBase accessToken 失败，请确认控制台已开启匿名登录」。

### 1.3.1 若请求网关返回 403：先开通 HTTP 访问服务

403 常见原因：**HTTP 访问服务未开启** 或 **网关未放行该云函数**。

1. 打开云开发控制台 → **环境管理** → 找到 **「HTTP 访问服务」** 或 **「API 网关」** 相关入口。
2. 确认 **HTTP 访问服务已开启**，且允许通过 `https://{环境ID}.api.tcloudbasegateway.com` 调用云函数。
3. 若控制台有「云函数 HTTP 触发」或「网关路由」配置，确认 **admin_manager** 已加入可访问列表。

### 1.3.2 若仍 403：使用 Publishable Key

若已开通 HTTP 访问仍 403，可改用 **Publishable Key** 作为 Bearer（长期有效、可暴露在浏览器）：

1. 打开云开发控制台 → **环境管理** → **ApiKey 管理**。
2. 找到或新建 **Publishable Key**（发布密钥），复制其值（注意不是 Secret Key）。
3. 在项目 **`.env.development`** 中新增：`VITE_CLOUDBASE_PUBLISHABLE_KEY=你的PublishableKey`
4. 保存后**重启** `npm run dev`，再试登录。

配置后，请求将优先使用 Publishable Key 作为 Bearer。

### 1.4 配置 CORS（若控制台有该项）

若在 HTTP 触发 / API 网关处有 **CORS** 配置，请增加：

- `http://localhost:3006`（与 1.2 中本地域名一致）
- 生产环境的前端域名

保存后，云函数才能被本地/生产页面请求到。

### 1.4.1 若登录报「Unknown action」：云函数需解析 event.body

使用 **HTTP 访问服务（默认域名）** 时，网关把 POST 请求体放在 **`event.body`（字符串）** 里，不会自动变成 `event.action` / `event.data`。若 admin_manager 只读 `event.action`，会得到 `undefined` 并返回 "Unknown action"。

**处理方式**：在 **admin_manager 云函数入口**（小程序/云开发项目里）对 `event` 做统一解析，兼容两种调用方式。在 `main` 或入口里、**最先**执行下面逻辑（在原有 `action = event.action` 之前）：

```js
// 兼容「HTTP 访问服务」：请求体在 event.body（字符串）
if (event.body !== undefined && event.body !== null) {
  const parsed = typeof event.body === 'string' ? JSON.parse(event.body) : event.body
  event.action = parsed.action
  event.data = parsed.data || {}
}
// 此后照常使用 event.action、event.data
```

保存后**重新部署** admin_manager，再试后台登录。

### 1.5 确保有一个可登录的管理员
1. 云开发控制台 → **数据库** → 打开 **users** 集合。
2. 找一条要当管理员的用户（或新建一条），必须包含：
   - **phone**：手机号（用来登录）
   - **password**：密码（和云函数里校验方式一致，一般是加密存的）
   - **is_admin**：设为 **true**（只有为 true 才能用 admin_login 登录后台）
3. 记住这个**手机号**和**密码**，后面在本地登录要用。

### 1.6 确认云函数和集合存在
- 云函数 **admin_manager** 已部署、已发布。
- 数据库里至少有：`users`、`admin_sessions`（其他集合按业务需要建）。

## 第二步：本地项目配置

### 2.1 填云函数网关地址
在项目根目录打开 **`.env.development`**，确认已有（或新增）云函数网关地址（不要引号、不要多余空格）：

```env
# 云函数 admin_manager 网关地址（环境 ID 为 cloud1-1gwuiims1ab650b6 时可直接用下面这行）
VITE_CLOUDBASE_HTTP_URL=https://cloud1-1gwuiims1ab650b6.api.tcloudbasegateway.com/v1/functions/admin_manager
```

若你的环境 ID 不同，把 `cloud1-1gwuiims1ab650b6` 换成你的环境 ID 即可。保存后需**重启** `pnpm dev`。

### 2.2 安装依赖（若还没装）
在项目目录下执行：

```bash
cd c:\Users\Administrator\Desktop\art-lnb-master
pnpm install
```

## 第三步：本地运行并登录

### 3.1 启动项目
```bash
pnpm dev
```

浏览器一般会自动打开 `http://localhost:3006`（或终端里提示的地址；端口以 `.env` 中 `VITE_PORT` 为准）。

### 3.2 登录
- **手机号**：填 1.3 里那个管理员的 **phone**。
- **密码**：填该用户的 **password**（和云库里一致；若云函数用加密，这里填加密前的明文密码，以你云函数逻辑为准）。
- 点 **登录**。

### 3.3 看是否对接成功
- 能进后台、左侧出现 **「儿童视力管理」** 菜单，下面有 8 个子菜单（用户管理、学校/班级字典等），说明**本地对接已成功**。
- 再点进「用户管理」等页面，能正常拉列表、不报错，即接口也正常。

## 第四步：常见问题排查

| 现象 | 处理 |
|------|------|
| 提示「未配置 VITE_CLOUDBASE_HTTP_URL」 | 检查 `.env.development` 是否写了 `VITE_CLOUDBASE_HTTP_URL`，改完后**关掉 dev 再重新执行 `pnpm dev`**。 |
| 登录一直转圈或报错 | 看浏览器 **F12 → Network**，点失败的请求看状态码和返回；再看 **Console** 有无 CORS 报错。 |
| CORS / 跨域报错 | 回第一步 **1.2 安全域名**，确认已添加 `http://localhost:3006`（或你实际端口）；若有 CORS 配置也需加上该来源。 |
| 提示非管理员 / 登录失败 | 检查 users 里该条数据的 **is_admin** 是否为 **true**，**phone**、**password** 是否和云函数校验逻辑一致。 |
| 404 / 接口地址错 | 确认 `.env.development` 里 `VITE_CLOUDBASE_HTTP_URL` 为 `https://{环境ID}.api.tcloudbasegateway.com/v1/functions/admin_manager`。 |

---

## 小结：本地对接 4 件事

1. **云开发**：使用网关地址 `https://{环境ID}.api.tcloudbasegateway.com/v1/functions/admin_manager`（无需在控制台单独找「HTTP 触发」）。
2. **安全域名**：在控制台「安全」/「环境配置 → 安全配置」里添加 `http://localhost:3006`（及生产域名）；并开启**匿名登录**。
3. **本地**：在 `.env.development` 里配置 `VITE_CLOUDBASE_HTTP_URL=上述网关地址`，保存后重启 `pnpm dev`。
4. **验证**：用 `is_admin: true` 的管理员手机号+密码登录，能进「儿童视力管理」即对接成功。

完成以上步骤即可在本地完成对接；之后部署到服务器只需在 `.env.production` 中配置同样的 `VITE_CLOUDBASE_HTTP_URL`（及生产环境安全域名），再构建部署即可。
