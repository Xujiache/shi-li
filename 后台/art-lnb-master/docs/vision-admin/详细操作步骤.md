# 儿童视力管理后台 — 详细操作步骤

本文档按顺序说明：如何配置环境、云开发侧准备、本地运行与登录、以及可选部署。

---

## 一、本地环境准备

### 1.1 安装 Node.js 与 pnpm
- 安装 **Node.js 18+**（推荐 LTS）：https://nodejs.org/
- 安装 pnpm（如未安装）：
  ```bash
  npm install -g pnpm
  ```

### 1.2 克隆/打开项目并安装依赖
```bash
# 进入项目目录（你当前是 art-lnb-master）
cd c:\Users\Administrator\Desktop\art-lnb-master

# 安装依赖
pnpm install
```

---

## 二、配置云函数 HTTP 地址（必须）

后台通过 HTTP 直连云函数 `admin_manager`，必须配置其访问地址。

### 2.1 获取 HTTP 触发 URL
1. 打开 **微信开发者工具**，进入你的小程序项目（含云开发）。
2. 点击左侧 **云开发** → 进入云开发控制台。
3. 在控制台左侧选择 **云函数**，找到并点击 **admin_manager**。
4. 进入 **触发器** 或 **HTTP 触发** 配置：
   - 若尚未配置 HTTP 触发：点击「创建触发器」→ 类型选 **HTTP 触发**，路径建议填 `/admin_manager`，方法 **POST**。
   - 若已配置：记下完整 URL，形如：
     ```text
     https://xxxxxx.service.tcloudbase.com/admin_manager
     ```
     或你自定义的域名 + 路径。

### 2.2 配置 CORS（允许后台前端域名）
在云函数/API 网关的 CORS 配置中，允许你的后台前端访问：
- 本地开发：可先允许 `http://localhost:5173`（或你实际用的端口）。
- 正式环境：填写后台前端域名，例如 `https://admin.yourdomain.com`。

### 2.3 在项目中填写环境变量
在项目根目录下编辑：

- **开发环境**：`.env.development`
- **生产环境**：`.env.production`

在文件中找到或新增一行，把上面得到的 URL 填进去（不要有多余空格或引号）：

```env
# 云函数 admin_manager 的 HTTP 触发地址
VITE_CLOUDBASE_HTTP_URL=https://xxxxxx.service.tcloudbase.com/admin_manager
```

保存后，**重启一次开发服务器**（见下文「本地运行」），环境变量才会生效。

## 三、云开发侧准备（云函数与数据库）

### 3.1 部署云函数
在微信开发者工具或云开发控制台中，确保以下云函数已部署并发布：
- **admin_manager**（B 端后台必选）
- 若 C 端小程序在用：`user_manager`、`profile_manager`、`data_manager`、`appointment_manager` 等也需按需部署。

### 3.2 确认云数据库集合
在云开发控制台 → **数据库** 中，确认存在以下集合（没有则新建）：
- `users`
- `children`
- `school_classes`
- `banners`
- `appointment_items`
- `appointment_schedules`
- `appointment_records`
- `checkup_records`
- `admin_sessions`

### 3.3 创建可登录后台的管理员账号
1. 在 **数据库** → **users** 集合中，找到要作为管理员的用户文档（或新建一条）。
2. 该文档必须包含：
   - `phone`：手机号（用于登录）
   - `password`：密码（与云函数中密码存储方式一致，一般为加密后）
   - **`is_admin`：设为 `true`**（只有 `is_admin === true` 的账号才能通过 `admin_login` 登录后台）
3. 若云函数对密码做了加密（如 bcrypt），新加的管理员密码需用同样规则加密后写入 `password` 字段。

## 四、本地运行与登录

### 4.1 启动开发服务器
```bash
cd c:\Users\Administrator\Desktop\art-lnb-master
pnpm dev
```

浏览器会打开本地地址（一般为 `http://localhost:5173`）。

### 4.2 登录后台
1. 在登录页：
   - **手机号**：填上面在 `users` 里设为管理员的 `phone`。
   - **密码**：填该用户的实际密码（与云库中加密前的密码一致，具体以你云函数校验逻辑为准）。
2. 点击登录。
3. 登录成功后会自动跳转，左侧菜单会出现 **「儿童视力管理」**，其下包括：
   - 用户管理
   - 学校/班级字典
   - 首页轮播管理
   - 预约项目
   - 预约排班
   - 预约记录
   - 孩子档案
   - 检测记录

### 4.3 若登录失败
- **提示「未配置 VITE_CLOUDBASE_HTTP_URL」**  
  检查 `.env.development` 中是否已正确填写 `VITE_CLOUDBASE_HTTP_URL`，保存后重启 `pnpm dev`。
- **提示「非管理员」或登录被拒绝**  
  检查该用户在 `users` 集合中 `is_admin` 是否为 `true`。
- **网络/CORS 报错**  
  确认云函数 HTTP 触发的 CORS 已允许当前前端地址（如 `http://localhost:5173`）。
- **控制台有 401/403**  
  确认云函数已部署、HTTP 路径与 `.env` 中 URL 一致（含路径，如 `/admin_manager`）。

## 五、生产构建与部署（可选）

### 5.1 构建
```bash
cd c:\Users\Administrator\Desktop\art-lnb-master
pnpm build
```
产物在项目根目录下的 **`dist`** 文件夹。

### 5.2 配置生产环境变量
构建前请确认 **`.env.production`** 中已填写生产环境的云函数地址：
```env
VITE_CLOUDBASE_HTTP_URL=https://你的生产环境云函数HTTP地址/admin_manager
```

### 5.3 部署到服务器
将 `dist` 目录整体上传到你的 Web 服务器（Nginx、Apache、或静态托管），并配置：
- 根目录指向 `dist`。
- 若使用前端路由 history 模式，需配置服务器把所有路径回退到 `index.html`（避免刷新 404）。

部署后，在云函数/网关的 CORS 中增加你的生产后台域名（如 `https://admin.yourdomain.com`）。

## 六、步骤速查表

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 安装 Node 18+、pnpm，执行 `pnpm install` | 本地环境 |
| 2 | 云开发控制台为 `admin_manager` 配置 HTTP 触发，记下 URL | 获取直连地址 |
| 3 | 配置 CORS 允许前端域名/本地 localhost | 避免跨域报错 |
| 4 | 在 `.env.development` 填写 `VITE_CLOUDBASE_HTTP_URL` | 开发环境必配 |
| 5 | 部署云函数，确认数据库集合存在 | 云开发侧 |
| 6 | 在 `users` 中设置一个 `is_admin: true` 的管理员 | 用于登录 |
| 7 | `pnpm dev` 启动，用该管理员手机号+密码登录 | 验证后台 |
| 8 | （可选）`.env.production` 配置后 `pnpm build`，部署 dist | 生产发布 |

按上述顺序做完 1～7，即可在本地完成「开发 + 对接云开发」并登录使用后台；需要上线时再按第 8 步做生产构建与部署。
